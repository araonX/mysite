Git之路在此开启
一、	创建Git版本库（repository）
1、	创建一个空目录
2、	通过git init 命令将这个目录变成Git可以管理的仓库(repository).
注:所有的版本控制系统，只能跟踪文本文件的改动，比如txt文件，网页，所有的程序代码等等。而图片、视频这些二进制文件虽然也能由版本控制系统管理，但是没办法跟踪文件的变化，只能把二进制文件每次改动串起来，也就是只知道图片从100KB改成了200KB，但是到底改了啥，系统控制系统不知道。
二、	提交Git版本库中的文件
1、	在创建的版本库中创建一个文件（如:git.py）.
2、	使用git add 告诉Git，把文件添加到仓库  git add git.py
3、	使用git commit 告诉 Git，把文件提交到仓库  git commit –m “add git.py” –m参数允许在提交的同时插入本次的提交说明。
注：Git添加文件需要add 和 commit两步其原因是commit可以一次提交很多文件，所以，你可以多次add不同的文件.
git add file1.txt
git add file2.txt file3.txt
git commit –m “add 3 files”
三、	查看仓库中文件状态
1、	git  status 查看当前仓库的状态。
2、	git  diff file 可以对比出文件具体作何更改。(该文件未进行add 操作，若已经git add,则无法对比出差异。)
